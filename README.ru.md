# Прототип PQC CI/CD-конвейера

Язык: [English](README.md) | Русский

Открытый прототип CI/CD-конвейера поставки программного обеспечения с поддержкой классических, гибридных и постквантовых криптографических профилей.

Проект демонстрирует, как выбор криптографического профиля, генерация TLS-сертификатов, развёртывание TLS-сервера и TLS-клиента, интеграционное тестирование, сбор метрик и формирование отчётов могут быть встроены в автоматизированный процесс поставки ПО.

## Назначение

Прототип разработан в рамках выпускной квалификационной работы по теме:

> Разработка конвейера поставки программного обеспечения с поддержкой постквантовой криптографии.

Цель проекта — проверить не отдельный криптографический примитив, а полный инженерный цикл интеграции криптографических профилей в воспроизводимый CI/CD-процесс.

Конвейер поддерживает:

* классический профиль TLS 1.3;
* гибридный профиль TLS 1.3 с классическим и постквантовым установлением ключа;
* максимально поддерживаемый постквантовый TLS-профиль.

## Технологический стек

* Ubuntu 24.04 LTS
* Docker и Docker Compose
* GitHub Actions
* Bash
* Python 3
* OpenSSL 3.5.6

OpenSSL собирается внутри Docker-образа, чтобы эксперимент не зависел от версии OpenSSL, установленной в операционной системе пользователя.

## Криптографические профили

Описание профилей хранится в каталоге `configs/profiles/`.

| Профиль     | Назначение                                                  | Группа обмена ключами | Тип сертификата |
| ----------- | ----------------------------------------------------------- | --------------------- | --------------- |
| `classical` | Базовый классический профиль TLS 1.3                        | `X25519`              | ECDSA / P-256   |
| `hybrid`    | Переходный профиль с классическим и PQC-установлением ключа | `X25519MLKEM768`      | ECDSA / P-256   |
| `pqc`       | Максимально поддерживаемый PQC-профиль                      | `MLKEM768`            | ECDSA / P-256   |

Профиль `pqc` использует постквантовый механизм установления ключа ML-KEM, но сохраняет классический сертификат ECDSA для стабильности TLS/X.509-сценария. Полная поддержка PQC-сертификатов на основе ML-DSA или SLH-DSA может быть добавлена как дальнейшее расширение.

## Быстрый запуск

Сборка Docker-образов:

```bash
make build
```

Запуск одного профиля:

```bash
make run-classical
make run-hybrid
make run-pqc
```

Запуск всех профилей и формирование общего отчёта:

```bash
make run-all
```

Полная локальная проверка проекта:

```bash
make check
```

Просмотр возможностей OpenSSL/PQC внутри Docker-образа:

```bash
make crypto-capabilities
```

Также доступны прямые команды запуска скриптов:

```bash
./scripts/build/build-images.sh
./scripts/run-pipeline.sh classical
./scripts/run-pipeline.sh hybrid
./scripts/run-pipeline.sh pqc
./scripts/run-all.sh
```

Результаты выполнения создаются в каталоге `artifacts/`.

Основные файлы:

```text
artifacts/metrics/raw-classical.csv
artifacts/metrics/raw-hybrid.csv
artifacts/metrics/raw-pqc.csv

artifacts/metrics/stages-classical.csv
artifacts/metrics/stages-hybrid.csv
artifacts/metrics/stages-pqc.csv

artifacts/metrics/runtime-classical.csv
artifacts/metrics/runtime-hybrid.csv
artifacts/metrics/runtime-pqc.csv

artifacts/reports/summary-classical.md
artifacts/reports/summary-hybrid.md
artifacts/reports/summary-pqc.md

artifacts/reports/summary-all.md
artifacts/reports/summary-all.csv
artifacts/reports/summary-all.json
```

Общие отчёты сравнивают все профили по success rate, времени TLS-рукопожатия, длительности стадий pipeline, сэмплированному потреблению ресурсов контейнером, согласованной группе обмена ключами и размерам сертификатов/ключей.

## Собираемые метрики

Конвейер собирает:

- статус и длительность TLS-рукопожатия;
- ожидаемую и фактическую группу обмена ключами;
- способ подтверждения группы;
- наблюдаемую версию TLS;
- код завершения OpenSSL и сообщение об ошибке;
- длительность отдельных стадий pipeline;
- общую длительность запуска профиля;
- сэмплированное потребление CPU и памяти TLS-сервером во время тестов;
- снимок сетевого ввода-вывода сервера;
- размеры серверного ключа, сертификата и сертификатной цепочки;
- размеры клиентских и серверных логов;
- сведения о версиях OpenSSL, Docker и Docker Compose;
- идентификаторы Docker-образов.

Сгенерированные SVG-графики визуализируют:

- среднее время TLS-рукопожатия;
- диапазон времени TLS-рукопожатий;
- общую длительность pipeline;
- длительность стадии интеграционного тестирования;
- максимальное сэмплированное потребление CPU сервером;
- максимальное сэмплированное потребление памяти сервером;
- размер сертификатной цепочки.

## Криптографическая проверка

Каждый профиль ограничивает TLS-согласование одной настроенной группой:

* `classical`: `X25519`;
* `hybrid`: `X25519MLKEM768`;
* `pqc`: `MLKEM768`.

Если OpenSSL выводит согласованный временный ключ, парсер использует это значение напрямую. Если OpenSSL не выводит временный ключ, но соединение TLS 1.3 успешно устанавливается с единственной разрешённой группой и сертификат проходит проверку, конвейер записывает настроенную группу с признаком подтверждения:

```text
restricted-single-group
```

Это делает проверку профиля воспроизводимой даже при изменении формата вывода OpenSSL CLI между версиями.

## GitHub Actions

В репозитории есть workflow GitHub Actions:

```text
.github/workflows/pipeline.yml
```

Workflow проверяет конфигурации профилей, собирает Docker-образы, запускает pipeline и сохраняет логи, метрики и отчёты как CI-артефакты.

Он запускается при:

* `push`;
* `pull_request`;
* ручном запуске через `workflow_dispatch`.

При ручном запуске можно выбрать `all`, `classical`, `hybrid` или `pqc`.

## Безопасность и ограничения

Сгенерированные приватные ключи, сертификаты, логи, метрики и отчёты являются runtime-артефактами и не должны коммититься в Git.

Текущие ограничения:

* тестовый стенд использует `openssl s_server` и `openssl s_client` вместо полноценного прикладного сервера;
* профиль `pqc` использует установление ключа ML-KEM, но сохраняет классический сертификат ECDSA;
* метрики CPU и памяти пока не собираются как самостоятельные поля отчёта;
* проект является исследовательским прототипом и не предназначен для промышленного криптографического развёртывания.
